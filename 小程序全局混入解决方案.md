# 微信小程序全局 Mixins 实现思路与实践

## 一、背景与需求

在微信小程序开发中，我们经常会遇到需要在多个页面中复用相同逻辑的场景，比如：
- 统一的页面生命周期处理（如日志记录、埋点统计）
- 全局的页面返回事件处理
- 通用的页面方法（如数据格式化、工具方法）

传统的做法是在每个页面中重复编写相同的代码，这不仅增加了代码量，也提高了维护成本。而微信小程序本身并不像 Vue 那样原生支持 mixins 功能，因此我们需要通过重写 `Page` 函数来实现全局 mixins 的功能。

## 二、实现思路

### 核心思想

通过拦截并重写微信小程序的 `Page` 构造函数，在页面注册时自动将全局 mixins 的逻辑合并到每个页面配置中。

### 实现步骤

1. **保存原始的 Page 函数**
2. **重写 Page 函数**，在函数内部：
   - 合并全局 mixins 的生命周期函数
   - 合并全局 mixins 的数据和方法
   - 处理优先级问题（页面自身配置优先于 mixins）
3. **调用原始 Page 函数**完成页面注册

## 三、核心代码实现

### 1. 创建全局 Mixins 文件

首先，我们需要创建一个全局 mixins 文件 `mixins/global.js`：

```javascript
// mixins/global.js
module.exports = {
    // 生命周期函数 - 页面加载
    onLoad(options) {
      console.log('global onLoad', options);
    },
    
    // 生命周期函数 - 页面显示
    onShow() {
      console.log('global onShow');
    },
    
    // 生命周期函数 - 页面隐藏
    onHide() {
      console.log('global onHide');
    },
    
    // 生命周期函数 - 页面卸载
    onUnload() {
      console.log('global onUnload');
      // 获取当前页面栈
      const pages = getCurrentPages();
      if (pages.length > 1) {
        const currentPage = pages[pages.length - 1];
        // 获取当前页面的路由信息
        const routeInfo = {
          route: currentPage.route, // 页面路径
          options: currentPage.options || {}, // 页面参数
          timestamp: Date.now() // 时间戳，用于标识
        };
        
        // 通过 EventChannel 传递路由信息（如果是从 navigateTo 跳转过来的）
        try {
          const channel = this.getOpenerEventChannel && this.getOpenerEventChannel();
          if (channel && channel.emit) {
            channel.emit('navPageBackEvent', routeInfo);
            console.log('通过 EventChannel 发送路由信息:', routeInfo);
          }
        } catch (e) {
          // EventChannel 不可用时，静默失败
        }
      }
    },
    
    // 全局混入的方法（所有页面可直接调用）
    globalMethod() {
      console.log('这是全局混入的方法');
    }
};
```

### 2. 在 app.js 中重写 Page 函数

在 `app.js` 文件的最开始（必须在所有页面文件引入之前），重写 `Page` 函数：

```javascript
// app.js
// 引入全局 mixins
const globalPageMixin = require('./mixins/global.js');

// 保存原始的 Page 函数
const originPage = Page;

// 重写 Page 函数
Page = function(config) {
  // 合并全局混入逻辑与页面自身配置
  
  // 1. 合并生命周期函数（如 onLoad、onShow、onUnload 等）
  const mixedLifetimes = {};
  
  // 遍历全局 mixins 的所有属性
  Object.keys(globalPageMixin).forEach(key => {
    // 如果是函数类型，则认为是生命周期函数或方法
    if (typeof globalPageMixin[key] === 'function') {
      mixedLifetimes[key] = function(...args) {
        // 优先执行全局 mixins 的逻辑
        globalPageMixin[key].apply(this, args);
        // 再执行页面自身的逻辑（如果页面有定义）
        if (config[key] && typeof config[key] === 'function') {
          config[key].apply(this, args);
        }
      };
    }
  });
  
  // 2. 合并数据和方法（页面自身配置优先级更高）
  // 先合并全局 mixins，再合并页面配置，最后用处理过的生命周期覆盖
  const newConfig = {
    ...globalPageMixin,      // 全局 mixins 的配置
    ...config,               // 页面自身的配置（会覆盖 mixins 中的同名属性）
    ...mixedLifetimes        // 处理过的生命周期函数（确保覆盖）
  };
  
  // 调用原始的 Page 函数完成页面注册
  originPage(newConfig);
}

// 正常的 App 配置
App({
  globalData: {
    // ... 全局数据
  },
  onLaunch: function () {
    // ... 启动逻辑
  },
  onShow: function (options) {
    // ... 显示逻辑
  }
})
```

## 四、关键点解析

### 1. 生命周期函数的合并策略

生命周期函数采用**链式调用**的方式：
- 先执行全局 mixins 中的生命周期函数
- 再执行页面自身定义的生命周期函数

这样可以确保：
- 全局逻辑先执行（如日志记录、埋点）
- 页面特定逻辑后执行（如页面数据初始化）

### 2. 优先级处理

合并顺序决定了优先级：
```javascript
const newConfig = {
  ...globalPageMixin,      // 第一层：全局 mixins
  ...config,               // 第二层：页面配置（覆盖 mixins）
  ...mixedLifetimes        // 第三层：处理过的生命周期（最终覆盖）
};
```

- **数据和方法**：页面配置优先于 mixins
- **生命周期函数**：通过 `mixedLifetimes` 特殊处理，确保链式调用

### 3. 页面返回事件处理

在 `onUnload` 生命周期中，我们实现了页面返回时的事件通知机制：

```javascript
onUnload() {
  const pages = getCurrentPages();
  if (pages.length > 1) {
    const currentPage = pages[pages.length - 1];
    const routeInfo = {
      route: currentPage.route,
      options: currentPage.options || {},
      timestamp: Date.now()
    };
    
    // 通过 EventChannel 发送事件
    const channel = this.getOpenerEventChannel && this.getOpenerEventChannel();
    if (channel && channel.emit) {
      channel.emit('navPageBackEvent', routeInfo);
    }
  }
}
```

## 五、使用示例

### 1. 普通页面使用

页面无需任何特殊处理，mixins 会自动生效：

```javascript
// pages/example/index.js
Page({
  data: {
    message: 'Hello World'
  },
  
  onLoad(options) {
    // 全局 mixins 的 onLoad 会先执行
    // 然后执行这里的 onLoad
    console.log('页面 onLoad');
  },
  
  onCustomMethod() {
    // 可以直接调用全局混入的方法
    this.globalMethod();
  }
});
```

### 2. 接收页面返回事件

如果需要在页面返回时接收路由信息，可以在跳转时设置事件监听：

```javascript
// 跳转页面
wx.navigateTo({
  url: '/pages/detail/detail',
  events: {
    // 监听页面返回事件
    navPageBackEvent: function(routeInfo) {
      console.log('接收到返回页面的路由信息:', routeInfo);
      // routeInfo 包含: route, options, timestamp
      // 可以在这里处理返回逻辑，如刷新数据等
    }
  }
});
```

## 六、优势与注意事项

### 优势

1. **代码复用**：全局逻辑只需定义一次，所有页面自动继承
2. **统一管理**：便于统一处理日志、埋点、错误处理等
3. **无侵入性**：页面代码无需修改，自动生效
4. **灵活扩展**：可以轻松添加新的全局功能

### 注意事项

1. **执行顺序**：必须在 `app.js` 的最开始重写 `Page`，确保在所有页面文件引入之前执行
2. **命名冲突**：注意避免 mixins 中的方法名与页面方法名冲突（页面配置会覆盖 mixins）
3. **生命周期顺序**：全局 mixins 的生命周期先于页面自身的生命周期执行
4. **性能考虑**：虽然 mixins 会增加一些代码，但影响微乎其微

## 七、扩展建议

### 1. 支持多个 Mixins

可以扩展为支持多个 mixins：

```javascript
const mixins = [
  require('./mixins/global.js'),
  require('./mixins/logger.js'),
  require('./mixins/analytics.js')
];

Page = function(config) {
  // 合并多个 mixins...
}
```

### 2. 条件性 Mixins

可以根据条件决定是否应用某个 mixin：

```javascript
const globalPageMixin = require('./mixins/global.js');
const debugPageMixin = require('./mixins/debug.js');

Page = function(config) {
  const mixins = [globalPageMixin];
  if (api.debug) {
    mixins.push(debugPageMixin);
  }
  // 合并逻辑...
}
```

### 3. Mixins 配置化

可以通过配置文件管理 mixins：

```javascript
// config/mixins.js
module.exports = {
  global: require('./mixins/global.js'),
  logger: require('./mixins/logger.js'),
  // ...
};
```

## 八、总结

通过重写 `Page` 函数实现全局 mixins，是一种优雅的代码复用方案。它让我们能够：

- ✅ 统一管理全局逻辑
- ✅ 减少代码重复
- ✅ 提高代码可维护性
- ✅ 实现页面间通信

这种方案已经在多个小程序项目中得到验证，是一种成熟且实用的技术方案。希望本文能够帮助你在小程序开发中更好地组织代码结构。
